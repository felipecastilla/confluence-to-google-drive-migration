Vitel : Arquitectura de alto nivel

Created by Felipe Castilla, last modified on Jan 06, 2023

El sistema es una aplicación Cliente - Servidor.

Cabe notar que este no es un thin client, por el contrario, el cliente juega un rol fundamental en la integración con varios dispositivos como: Escaner y POS.

Al momento de escribir este documento aún desconocemos si es el cliente quien realiza la impresión o es el servidor.

Diagrama de contexto:

Inicialmente vemos una aplicación corriendo sobre un sistema operativo Windows en un navegador Chrome. Soy especifico con el sistema operativo por el simple hecho que los drivers de los dispositivos solo son compatible con dicho S.O.

El navegador Chrome deberá ser iniciado en kiosk mode y full screen de alguna forma una vez Windows inicie.

Idealmente se creará un script .bat para Windows que permita realizar la configuración de inicio de Chrome en dicho modo apuntando a la URL de la aplicación.

Ejemplo de .bat: https://gist.github.com/kstracq/c40d1b94ef2c9583bf0b0b4dcadbacdb

Command-line swtiches de Chrome: https://peter.sh/experiments/chromium-command-line-switches/

Cliente

Aquí estará la aplicación y demás componentes que deben ejecutarse al iniciar el sistema operativo

Deberá estar provisionado con:

  • Un navegador Chrome

  • La aplicación de frontend que se descargará desde el servidor corriendo sobre un navegador el Chrome

  • El agente que se comunicará con el POS de Transbank

  • Los drivers del Scanner de Codigo de barra

  • Los driver de la impresora

Los drivers de la impresora aún no estamos seguros que los vayamos a instalar en el Cliente o es el server quien emite el evento de impresión.

En particular este provisionamiento deberá ser suministrado por un script que reposará en el repositorio del proyecto.

Servidor

En un Ubuntu Server que estará provisionado con Docker, se ejecutarán los siguientes containers:

  • Nginx: para servir los archivos estaticos de la app y servir como proxy de los endpoints

  • NodeJS: para tener los endpoints que servirán la información a la aplicación de frontend

  • MariaDB: donde reposaran los datos de la aplicación

El servidor deberá además tener conectividad con el Web Services de Softland, para la creación de la Factura Electronica.
